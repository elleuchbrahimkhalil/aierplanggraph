### Maven Parent overrides

Due to Maven's design, elements are inherited from the parent POM to the project POM.
While most of the inheritance is fine, it also inherits unwanted elements like `<license>` and `<developers>` from the parent.
To prevent this, the project POM contains empty overrides for these elements.
If you manually switch to a different parent and actually want the inheritance, you need to remove those overrides.

## Airflow setup on Windows (Docker)

This project now includes a local Apache Airflow setup using Docker Compose.

### 1) Environment variables

Variables are configured in `.env.airflow`:

- `AIRFLOW_UID=50000`
- `AIRFLOW_GID=0`
- `AIRFLOW__CORE__LOAD_EXAMPLES=false`
- `_AIRFLOW_WWW_USER_USERNAME=admin`
- `_AIRFLOW_WWW_USER_PASSWORD=admin123`

You can change admin credentials in this file before startup.

### 2) Start Airflow

Run from the project root:

```powershell
docker compose --env-file .env.airflow -f docker-compose.airflow.yml up -d
```

Airflow UI will be available at:

- `http://localhost:8080`

Default login:

- username: `admin`
- password: `admin123`

### 3) Stop Airflow

```powershell
docker compose --env-file .env.airflow -f docker-compose.airflow.yml down
```

### 4) Included example DAG

An example DAG is available in `airflow/dags/hello_world_dag.py`.
